# 택배건수 집계 시스템 (sales_orders)

**Why:** 판매량 대시보드에서 정확한 택배건수를 표시하기 위해. 합배송 중복 제거와 본사소진/협찬 제외가 필수.

**How to apply:** 택배건수 데이터 업로드 시 반드시 이 로직을 따라야 함.

## 데이터 소스

1. **드림산업 (제조사 직접 배송)**
   - 1~4월: 주문번호 기준 (송장번호 없음)
   - 5~6월: 롯데택배 송장번호 기준 (합배송 중복 제거 필요)
   - 엑셀 파일: `드림산업_X월택배건_YYMMDD.xlsx`

2. **투민글로벌 (3PL 물류센터)**
   - 전 월: 오더코드(O20260101-xxxxx) 기준
   - 엑셀 파일: `발주조회_YYYYMMDDHHMMSS.xlsx`
   - **중요**: "발주 타입" 컬럼에서 "본사소진"과 "협찬" 제외 필수

## 제외 규칙 (매우 중요)

### 투민글로벌 제외 대상
1. **발주 타입이 "본사소진" 또는 "협찬"** ← 가장 중요!
2. 배송메모에 "증정", "사은품", "협찬" 포함
3. 상품명에 "증정", "사은품", "행사", "포장재", "협찬" 포함

### 실제 제외 건수 (2026년 1~6월)
- 1월: 24건 (본사소진 5건 + 협찬 19건)
- 2월: 12건 (본사소진 2건 + 협찬 10건)
- 3월: 0건
- 4월: 13건 (본사소진 6건 + 협찬 7건)
- 5월: 데이터 없음
- 6월: 데이터 없음
- **총 49건 제외**

## 중복 제거 규칙

### 합배송 처리 (송장번호 기준)
- 동일한 송장번호 = 택배 1건
- 5~6월 드림산업 데이터에 적용
- Python Set을 사용하여 중복 제거:
```python
tracking_map = {}
for row in data:
    tracking_no = row['송장번호']
    if tracking_no not in tracking_map:  # 중복 제거
        tracking_map[tracking_no] = {...}
```

### 주문번호/오더코드 기준
- 고유한 주문번호만 집계
- 1~4월 드림산업, 전월 투민글로벌에 적용

## sales_orders 테이블 구조

```
- order_date: 주문일 (YYYY-MM-DD)
- warehouse: '드림산업' 또는 '투민글로벌'
- barcode: 송장번호 또는 주문번호/오더코드
- order_number: 주문번호
- quantity: 1 (택배건수이므로 항상 1)
- is_set: false
- created_by: 'system_excel' 또는 'system_tumin'
```

1. **3월 드림산업**: 최상단 3행(최혜수, 김숙자, 최은경) 제외
2. **daily_sales 동기화**: sales_orders 삭제 시 daily_sales도 함께 삭제 필요

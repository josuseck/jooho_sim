반품/교환 엑셀 시트의 컬럼 데이터 형식이 과거와 최근에 다르다.

**바코드 컬럼 변화:**
- **과거 (2025.11~2026.3)**: 결제수단 (신용카드, 계좌이체 등)
- **최근 3개월 (2026.4~6)**: 제품 바코드 (LE10400001005, R008535120006 등)

**불량여부 컬럼 형식:**
- **엑셀 원본**: True/False (체크박스)
- **DB 저장 형식**: "경결함"/"미결함" (문자열)
- **변환 규칙**:
  - True → "경결함" (상품불량)
  - False → "미결함" (단순변심, 색상/사이즈 변경 등)

**업로드 주의사항:**
- 송정선 데이터 이후부터 업로드 (사용자 요청 기준점)
- 교환/반품 컬럼이 null인 행은 빈 행이므로 제외
- 최근 3개월 데이터만 바코드 컬럼이 제품 바코드로 정상 입력됨

**Why:** 시트 형식이 중간에 변경되었으며, 최근 데이터만 제품 바코드가 정확히 기록됨.

**How to apply:**
- 반품/교환 데이터 업로드 시 불량여부 True/False → 경결함/미결함 변환 필수
- 바코드 컬럼은 최근 3개월 데이터에서만 의미 있음
- 과거 데이터의 바코드 컬럼은 결제수단이므로 무시
